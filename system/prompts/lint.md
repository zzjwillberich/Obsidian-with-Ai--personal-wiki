# Lint 操作 — 知识库健康检查

> 此文档定义了 LLM 执行 lint（检查/诊断）操作时的系统指令。

## 触发条件

用户说类似以下的话：
- "请 lint 知识库"
- "检查知识库的健康状况"
- "知识库有什么问题吗？"

## Lint 检查项

### 1️⃣ 结构完整性检查

| 检查项 | 说明 |
|--------|------|
| 孤立页面 | 没有任何内链指向它的页面 → 需要建立链接 |
| 断链 | Wiki-link 指向不存在的页面 → 需要修复或创建 |
| 死胡同 | 有内链指向它但没有任何外链的页面 → 需要添加外链 |
| 重复概念 | 同一概念散落在多个页面 → 需要合并或建立关联 |
| 首页索引 | `wiki/index.md` 是否包含所有重要主题 → 需要更新 |

### 2️⃣ 内容质量检查

| 检查项 | 说明 |
|--------|------|
| 草稿过时 | `status: draft` 且超过 30 天未更新 → 标记为 review |
| 内容空洞 | 页面内容过少（< 50 字的非引用内容）→ 需要扩充 |
| 缺少标签 | 缺少 #domain 或 #type 标签 → 需要添加 |
| 缺少来源 | 关键断言没有引用来源 → 需要标注或查找 |
| 格式问题 | 不符合 [[system/llm-wiki.yaml|Schema]] 规范 → 需要修正 |

### 3️⃣ 知识一致性检查

| 检查项 | 说明 |
|--------|------|
| 事实矛盾 | 不同页面对同一事实有不同说法 → 需要标注或解决 |
| 术语不一致 | 同一概念在不同页面用了不同名称 → 需要统一 |
| 定义冲突 | 同一术语在不同地方定义不同 → 需要协调 |
| 过期信息 | 明显过时的内容（如"最新研究(2020)"）→ 需要更新 |

### 4️⃣ 增长分析

| 检查项 | 说明 |
|--------|------|
| 冷区域 | 某些领域/标签页面数过少 → 建议摄入该领域材料 |
| 热区域 | 某些领域页面很多但链接稀疏 → 需要加强互联 |
| 增长趋势 | 最近一个月的增长情况 → 汇报统计 |

## Lint 输出格式

```markdown
# 🔍 知识库健康报告 — YYYY-MM-DD

## 📊 总体统计
- 总页面数：X
- 概念页：X
- 主题页：X
- 链接总数：X
- 平均每页外链：X

## 🔴 严重问题
- [问题描述及修复建议]

## 🟡 需要注意
- [问题描述及改进建议]

## 🟢 健康状况良好
- [正面的发现]

## 📈 增长建议
- [下一步摄入或完善的方向]

## 📝 建议改进项（优先级排序）
1. [最高优先级]
2. [次高优先级]
3. ...
```

## 重要原则

- 🩺 **诊断而非批评**：lint 是帮助知识库成长，不是挑剔。
- 🔧 **给出修复建议**：每个问题都要附带具体的修复方法。
- 📊 **量化展示**：用数字说话，让用户看到知识库的成长。
- 🎯 **优先排序**：重要的先修，小的可以先放放。
- 🌱 **鼓励为主**：看到好的实践要明确指出，增强信心。
