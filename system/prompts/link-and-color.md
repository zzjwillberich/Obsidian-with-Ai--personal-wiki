# 链接与颜色组规范

> 全局规则：链接怎么连、颜色怎么分、标签怎么打。
> 每次创建/编辑页面时必须遵守。

---

## 1. 链接规范：强相关链接原则

### 核心理念

> **只连强相关，不画蜘蛛网。**

Wiki 链接的价值在于精准导航——读者沿着链接走，每一步都应该有**明确的知识依赖关系**。链接太多等于没链接，图谱中的线密集到看不清反而失去了意义。

### 判断标准

| 可以连 ✅ | 不要连 ❌ |
|-----------|-----------|
| A 是理解 B 的**必要前提**（如 `指针 → 智能指针`） | 只是标签层面有交集但内容独立（如 `指针` 和 `CMake`） |
| A 和 B 属于**同一个紧密体系**（如 `epoll → EventLoop → Reactor模式`） | 概念上沾边但彼此可以独立理解 |
| A 是 B 的**上游依赖**或**下游应用**（如 `TCP → socket编程`） | "都用了 C++""都在 wiki/concepts 里"这类弱关联 |
| A 和 B 是**父子/包含关系**的主题页面 | 跨越不同板块但共享一个宽泛标签 |

### 实际操作规则

1. **创建链接前先问**：删掉这个链接，读者会不会丢失理解某概念所必需的信息？如果不会，这条链接就不该存在。
2. **每页面出链控制在 3-8 条**（概念页），主题页可稍多。超过 10 条出链请重新审视。
3. **反向链接同样要求强相关**——被链接的页面往回连时，也要满足强相关标准。
4. **删除弱链接不手软**——lint 时发现"沾边就链"的应该主动断开。
5. **优先链向原子概念页**，而非宽泛主题页。

### 与旧规范的冲突说明

> ⚠️ 本规则**覆盖** CLAUDE.md 中「行为规范」第 1 条"链接第一"。
> 旧："每次创建或编辑页面，确保它与其他页面有双向链接。孤立页面是 bug。"
> 新：**每次创建或编辑页面，确保它与强相关页面有双向链接。无合适合适链接时不强行连——宁可暂时孤立，不要乱连。**

## 2. 颜色组规范：板块优先原则

### 核心理念

> **颜色按板块（领域）划分，不按编程语言划分。**

同一种编程语言内，网络编程和语法基础是完全不同的领域，应该有各自独立的颜色标识。一个节点在知识图谱中的颜色，应该让人一眼看出它属于哪个**知识板块**。

### 板块 → 颜色映射表

| 板块 | 图谱标签匹配 | 文件夹色 (CSS) | 说明 |
|------|-------------|---------------|------|
| 🔤 **C++ 语言核心** | `tag:#cpp-core` `tag:#cpp-grammar` | `#FD79A8` 樱花粉 | 指针、引用、RAII、智能指针、内存模型、编译过程 |
| 🌐 **网络编程** | `tag:#network` | `#FDCB6E` 金盏黄 | socket、epoll、TCP、HTTP、Reactor、muduo、零拷贝 |
| 🗄️ **数据库** | `tag:#db` | `#E17055` 陶土橙 | MySQL、SQL、事务、连接池 |
| ⚙️ **并发与系统** | `tag:#concurrency` | `#55EFC4` 薄荷绿 | 线程池、内存模型（系统视角） |
| 🔧 **工具链** | `tag:#tools` `tag:#build` | `#74B9FF` 天蓝 | CMake、Git、Debug/Release |
| 🎬 **创作/自媒体** | `tag:#creator` | `#E056A0` 洋红 | 选题、视频策划、内容素材 |
| 🧠 **成长策略** | `tag:#strategy` `tag:#career` `tag:#learning` | `#7C4FFF` 紫 | 职业规划、学习方法 |
| 📝 **英语/考试** | `tag:#english` `tag:#exam` | `#A29BFE` 薰衣草紫 | 四级备考、作文、方法论 |
| 🏗️ **系统配置** | `path:system` | `#7C8FA1` 石板灰 | CLAUDE.md 等，自身不做图谱颜色 |

### 操作规则

1. **新建页面时**：确定它属于哪个板块，打上对应的板块标签（而非笼统的 `#cpp`）。
2. **颜色组与文件夹色同步**：graph.json 中的颜色组和 folder-colors.css 中的文件夹色应使用相同的色值。
3. **板块标签优先于语言标签**：过去几乎所有技术页都打 `#cpp`，这是问题根源。板块标签才是颜色组的匹配依据。
4. **一个页面可以属于多个板块**——打多个板块标签即可，图谱中取第一个匹配的颜色组。

## 3. 标签规范：告别万能 #cpp

### 问题

当前绝大多数技术页面都打 `#cpp` 标签，导致：
- 图谱中所有 C++ 相关内容一种颜色，失去区隔
- 标签本身失去筛选意义（"#cpp 有 30 个页面"和"没标签"没区别）

### 规则

1. **标签必须是板块标签**——描述内容属于哪个知识领域，而非用的哪种编程语言。
2. **禁止把 `#cpp` 当万能标签**——语法页打 `#cpp-core`，网络页打 `#network`，数据库页打 `#db`。
3. **`#cpp` 降级为辅助标签**——仅在没有更精确的板块标签时使用（未来随板块标签完善逐步淘汰）。
4. **每页面 3-5 个标签为宜**：1-2 个板块标签 + 1-2 个内容标签 + 可选 1 个状态标签。

### 标签对照表（迁移指南）

| 旧标签组合 | 新标签组合 | 适用页面举例 |
|-----------|-----------|------------|
| `[cpp, cs]` | `[cpp-core, cs]` | 指针、引用、RAII |
| `[cpp, network, systems]` | `[network, systems]` | socket、epoll、TCP |
| `[cpp, network, muduo]` | `[network, muduo]` | EventLoop、Reactor |
| `[cpp, os, systems]` | `[concurrency, systems, os]` | 线程池、内存模型 |
| `[cpp, tools, build]` | `[tools, build]` | CMake |
| `[db, cs, backend]` | `[db, backend]` | MySQL |

## 4. Lint 检查项

Lint 模式执行时应额外检查：

- [ ] 是否存在弱相关链接（沾边就链）→ 标记建议删除
- [ ] 页面标签是否仍在使用万能 `#cpp` → 标记建议迁移
- [ ] 颜色组是否与板块匹配 → 标记不一致项
- [ ] 是否存在同一个颜色组覆盖了不同板块 → 标记建议拆分
- [ ] 页面出链数是否超过 10 条 → 标记审查
